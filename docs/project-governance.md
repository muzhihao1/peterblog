## 🏗️ 项目架构决策

### 1. 静态导出模式

- **决策**：使用 Next.js 静态导出（`output: 'export'`）
- **原因**：无需服务器，部署简单，性能最优
- **影响**：不支持动态 API 路由，所有数据构建时获取

### 2. Notion 作为 CMS

- **决策**：使用 Notion 数据库管理内容
- **原因**：可视化编辑，团队协作方便
- **影响**：需要处理 API 限制和故障情况

### 3. TypeScript 严格模式

- **决策**：启用 TypeScript 严格模式
- **原因**：提高代码质量，减少运行时错误
- **影响**：开发时需要更多类型定义
