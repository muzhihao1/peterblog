## 一、数据库迁移（立即执行）

### 执行顺序
```bash
# 1. 评论系统表（已有）
psql $DATABASE_URL < scripts/supabase-migration-comments.sql

# 2. 通知系统表
psql $DATABASE_URL < scripts/supabase-migration-notifications.sql

# 3. 数据分析表
psql $DATABASE_URL < scripts/supabase-migration-analytics.sql

# 4. 性能监控表
psql $DATABASE_URL < scripts/supabase-migration-monitoring.sql
```

### 迁移后验证
```sql
-- 检查表是否创建成功
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN (
  'notifications',
  'analytics_events',
  'analytics_sessions',
  'monitoring_metrics',
  'monitoring_alerts'
);
```

## 二、Supabase配置

### 1. 启用Realtime
在Supabase Dashboard中：
- Database → Replication
- 启用以下表的实时更新：
  - `comments`
  - `notifications`
  - `analytics_events`

### 2. 配置定时任务
在SQL Editor中创建定时任务：
```sql
-- 每小时清理过期数据
SELECT cron.schedule(
  'cleanup-analytics',
  '0 * * * *',
  'SELECT cleanup_old_analytics_data(90);'
);

-- 每天清理过期监控数据
SELECT cron.schedule(
  'cleanup-monitoring',
  '0 0 * * *',
  'SELECT cleanup_old_monitoring_data(30);'
);
```

## 五、部署步骤

### 1. 预览环境部署
```bash
# 创建预览分支
git checkout -b deploy/realtime-analytics-monitoring

# 提交环境变量更新
vercel env pull

# 部署到预览环境
vercel --prod
```

### 2. 监控部署状态
