# 博客图片存储方案

## 概述

本方案提供了一个完整的图片存储解决方案，主要使用 GitHub + jsDelivr CDN 的组合，实现免费、稳定、快速的图片托管服务。

## 方案对比

### 1. GitHub + jsDelivr CDN（推荐）
- **优点**：
  - ✅ 完全免费
  - ✅ 稳定可靠（GitHub 的稳定性）
  - ✅ 全球 CDN 加速
  - ✅ 版本控制
  - ✅ 与代码仓库集成
- **缺点**：
  - ❌ 单文件不能超过 100MB
  - ❌ 仓库总大小不能超过 1GB（推荐）
  - ❌ 不支持动态图片处理

### 2. Cloudinary（备用）
- **优点**：
  - ✅ 专业图片 CDN
  - ✅ 自动优化和转换
  - ✅ 25GB 免费存储 + 25GB/月流量
  - ✅ 支持动态裁剪、格式转换
- **缺点**：
  - ❌ 免费版有限制
  - ❌ 需要注册账号

### 3. Vercel Blob
- **优点**：
  - ✅ 与 Next.js 集成良好
  - ✅ 简单易用
  - ✅ 有免费额度
- **缺点**：
  - ❌ 相对较新，生态不够成熟
  - ❌ 免费额度有限

### 2. 目录结构

```
blog-assets/
├── projects/              # 项目图片
│   ├── personal-blog/
│   │   ├── thumbnail.jpg    # 缩略图 (800x600)
│   │   ├── screenshot-1.jpg # 截图
│   │   └── screenshot-2.jpg
│   ├── task-manager/
│   │   └── thumbnail.jpg
│   └── ...
├── posts/                # 博客文章图片
│   ├── 2024-01-01-title/
│   │   ├── cover.jpg
│   │   └── image-1.jpg
│   └── ...
├── books/                # 书籍封面
└── misc/                 # 其他图片
```

### 3. 图片命名规范

- 使用小写字母和连字符
- 缩略图：`thumbnail.jpg`
- 截图：`screenshot-1.jpg`, `screenshot-2.jpg`
- 避免使用中文和特殊字符

### 4. jsDelivr CDN URL 格式

```
https://cdn.jsdelivr.net/gh/[用户名]/blog-assets@main/[路径]/[文件名]

示例：
https://cdn.jsdelivr.net/gh/petermu/blog-assets@main/projects/personal-blog/thumbnail.jpg
```

### 5. 图片优化建议

#### 尺寸规范
- 项目缩略图：800x600
- 项目截图：1200x800
- 博客封面：1200x630（适合社交分享）
- 内容图片：最大宽度 1200px

#### 格式选择
- JPEG：适合照片和复杂图像
- PNG：适合图标和需要透明背景的图片
- WebP：更小的文件大小（需要考虑兼容性）

#### 压缩建议
- JPEG 质量：85-90%
- PNG：使用 TinyPNG 或类似工具
- 目标：缩略图 < 200KB，大图 < 500KB

### 6. 在 Notion 中使用

1. 上传图片到 GitHub 仓库
2. 获取 jsDelivr CDN 链接
3. 在 Notion 的 Thumbnail 字段中：
   - 选择 "Embed link"
   - 粘贴 CDN 链接

## Cloudinary 备用方案

### 配置步骤

1. 注册 Cloudinary 账号
2. 获取 API 凭证
3. 安装 SDK：`npm install cloudinary`
4. 配置环境变量：
   ```
   CLOUDINARY_CLOUD_NAME=your_cloud_name
   CLOUDINARY_API_KEY=your_api_key
   CLOUDINARY_API_SECRET=your_api_secret
   ```

### URL 格式

```
https://res.cloudinary.com/[cloud_name]/image/upload/[transformations]/[public_id]

示例（自动优化）：
https://res.cloudinary.com/your-cloud/image/upload/q_auto,f_auto,w_800/projects/personal-blog
```

## 最佳实践

1. **版本控制**：使用 Git 标签管理图片版本
2. **备份策略**：定期备份到其他云存储
3. **监控使用**：关注仓库大小和流量
4. **渐进式迁移**：先迁移新项目，再逐步迁移旧项目
5. **CDN 缓存**：利用 jsDelivr 的缓存机制，避免频繁更新

## 故障处理

### jsDelivr 无法访问
1. 检查仓库是否为公开
2. 确认文件路径正确
3. 等待 CDN 缓存更新（约 10 分钟）

### 图片加载慢
1. 检查图片大小是否过大
2. 使用图片优化工具压缩
3. 考虑使用 Cloudinary 的自动优化

### 超出 GitHub 限制
1. 清理不需要的图片
2. 创建新的图片仓库
3. 迁移到 Cloudinary 或其他服务

## 总结

推荐使用 GitHub + jsDelivr 作为主要方案，适合个人博客和中小型项目。如需要动态图片处理功能，可以结合使用 Cloudinary。这个方案实现了零成本、高可用、全球加速的图片存储服务。
