# 数据统计功能使用指南

本文档介绍博客的数据统计功能，包括实时统计、年度总结和内容表现仪表板。

## 📊 功能概述

博客提供了全面的数据统计和分析功能，帮助你了解内容表现和创作趋势：

1. **实时统计页面** (`/stats`) - 查看所有内容的实时统计数据
2. **年度总结页面** (`/year-in-review`) - 回顾年度创作成就
3. **数据仪表板** (`/dashboard`) - 内容表现分析和增长趋势
4. **统计 API** (`/api/statistics`) - 获取 JSON 格式的统计数据

## 🔧 核心组件

### 1. 统计数据模块

**位置**: `/lib/statistics.ts`

提供完整的博客统计数据收集功能：

- 文章统计（总数、字数、分类、标签）
- 项目统计（状态、技术栈）
- 书籍统计（阅读状态、评分）
- 工具统计（分类、评分）

### 2. 年度统计模块

**位置**: `/lib/statistics/year-statistics.ts`

专门处理年度数据分析：

- 月度发布分布
- 热门文章排行
- 写作连续性统计
- 年度里程碑计算

### 3. 统计页面组件

- `StatsClient` - 实时统计页面客户端组件
- `YearInReviewClient` - 年度总结交互组件
- `ContentDashboard` - 内容表现仪表板
- `YearSelector` - 年份选择器

## 📈 数据指标说明

### 文章相关指标

1. **总字数** - 所有文章的累计字数（中英文混合计算）
2. **平均字数** - 每篇文章的平均字数
3. **总阅读时间** - 基于 200-250 字/分钟的阅读速度计算
4. **月度分布** - 每月发布文章数量
5. **写作连续性** - 最长连续创作天数和当前连续天数

### 项目相关指标

1. **项目状态** - 进行中、已完成、其他
2. **技术栈使用** - 各技术栈在项目中的使用频率
3. **项目分类** - 不同类型项目的分布

### 书籍相关指标

1. **阅读进度** - 已读、在读、想读的书籍数量
2. **平均评分** - 已读书籍的平均评分
3. **总页数** - 所有书籍的累计页数

## 🎯 使用场景

### 1. 查看实时统计

访问 `/stats` 页面，可以看到：

- 各类内容的总数统计
- 分类和标签分布
- 热门内容排行
- 技术栈使用情况

### 2. 生成年度总结

访问 `/year-in-review` 页面，可以：

- 查看指定年份的创作数据
- 了解月度创作趋势
- 发现热门文章和标签
- 回顾年度里程碑

### 3. 监控内容表现

访问 `/dashboard` 页面，可以：

- 查看本月发布情况
- 监控内容增长率
- 分析分类表现
- 快速访问详细统计

## 🛠️ 技术实现

### 缓存机制

统计数据使用内存缓存，默认缓存时间为 1 小时：

```typescript
const CACHE_DURATION = 60 * 60 * 1000; // 1小时
```

### 数据更新

- 页面加载时自动获取最新数据
- 提供手动刷新按钮更新缓存
- API 端点支持强制刷新

### 性能优化

1. **延迟加载** - 统计组件使用动态导入
2. **并行计算** - 使用 Promise.all 并行获取各类数据
3. **按需渲染** - 图表和可视化组件按需加载

## 📝 扩展开发

### 添加新的统计指标

1. 在 `BlogStatistics` 接口中定义新字段
2. 在对应的统计函数中实现计算逻辑
3. 在展示组件中添加新指标的呈现

### 自定义可视化

1. 创建新的图表组件
2. 在统计页面中集成
3. 使用统计数据作为数据源

### API 集成

统计 API 返回标准 JSON 格式，可用于：

- 第三方工具集成
- 自动化报告生成
- 数据导出功能

## 🔍 常见问题

### 统计数据不准确？

检查：

1. Notion API 是否正常连接
2. 内容是否已发布
3. 缓存是否需要刷新

### 年度总结为空？

确认：

1. 选择的年份是否有内容
2. 文章日期字段是否正确
3. 数据获取是否成功

### 性能问题？

优化建议：

1. 调整缓存时间
2. 限制统计数据范围
3. 使用分页加载
