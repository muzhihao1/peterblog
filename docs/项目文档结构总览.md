## 文档分类说明

### 1. 项目级文档（/docs/）
- **用途**：项目管理、任务分配、进度跟踪
- **读者**：项目管理者、终端协调者
- **特点**：宏观视角、任务导向

### 2. 应用级文档（/my-blog/docs/）
- **用途**：技术实现、使用指南、维护手册
- **读者**：开发者、运维人员、内容创作者
- **特点**：技术细节、操作指南

## 维护建议

1. **新文档添加规则**：
   - 项目管理类 → `/docs/`
   - 技术文档类 → `/my-blog/docs/`
   - 按类别放入相应子目录

2. **文档命名规范**：
   - 使用中文或英文描述性名称
   - 避免使用版本号（使用 Git 管理版本）
   - 报告类文档可包含日期

3. **定期维护**：
   - 每月检查并归档过时文档
   - 保持 README.md 索引更新
   - 删除不再需要的临时文档

4. **文档质量**：
   - 每个文档应有明确的更新日期
   - 包含目标读者说明
   - 提供具体的操作步骤
