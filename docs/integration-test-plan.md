# 第二阶段集成测试计划

## 测试概述

本测试计划用于验证终端A（后端API）和终端B（前端功能）的集成是否正常工作。

## 测试前准备

### 1. 环境检查清单
- [ ] Supabase数据库已初始化
- [ ] 所有环境变量已配置
- [ ] GitHub OAuth已在Supabase配置
- [ ] 开发服务器可以正常启动
- [ ] 数据库连接测试通过

### 2. 测试账号准备
- 主测试账号：使用GitHub登录
- 游客测试：未登录状态
- 多账号测试：准备2-3个GitHub账号

## 测试场景

### 一、认证系统测试

#### 1.1 GitHub OAuth登录流程
```
测试步骤：
1. 点击导航栏"登录"按钮
2. 选择GitHub登录
3. 跳转到GitHub授权页面
4. 授权后回调到网站
5. 验证登录状态显示正确

预期结果：
- 成功登录后显示用户头像和名称
- 用户信息存储在Supabase
- 刷新页面后保持登录状态
```

#### 1.2 登出功能
```
测试步骤：
1. 在已登录状态下点击用户头像
2. 选择"退出登录"
3. 确认登出

预期结果：
- 成功登出，返回游客状态
- 清除所有认证cookie
- 无法访问需要登录的功能
```

### 二、点赞功能测试

#### 2.1 游客点赞
```
测试步骤：
1. 未登录状态访问文章
2. 点击点赞按钮

预期结果：
- 弹出登录提示Modal
- 点击"去登录"跳转到登录页
- 登录后返回原页面
```

#### 2.2 用户点赞/取消点赞
```
测试步骤：
1. 登录状态下访问文章
2. 点击点赞按钮（第一次）
3. 再次点击点赞按钮（取消）

预期结果：
- 第一次点击：爱心变红，数字+1
- 第二次点击：爱心变空，数字-1
- 动画效果流畅
- 数据正确保存到数据库
```

#### 2.3 点赞数据同步
```
测试步骤：
1. 用户A点赞文章
2. 用户B访问同一文章
3. 用户B看到的点赞数

预期结果：
- 用户B看到正确的点赞数
- 实时更新（如果实现了WebSocket）
```

### 三、收藏功能测试

#### 3.1 收藏文章
```
测试步骤：
1. 登录状态下点击收藏按钮
2. 访问个人中心-收藏列表

预期结果：
- 收藏成功，书签图标填充
- 收藏列表显示该文章
- 可以从列表跳转到文章
```

#### 3.2 批量管理收藏
```
测试步骤：
1. 收藏多篇文章（5-10篇）
2. 进入收藏管理页面
3. 选择多个收藏项
4. 点击"批量取消收藏"

预期结果：
- 批量选择功能正常
- 批量取消成功
- 列表实时更新
```

### 四、评论系统测试

#### 4.1 发表评论
```
测试步骤：
1. 登录状态下访问文章
2. 在评论框输入内容
3. 点击"发表评论"

预期结果：
- 评论成功发表
- 显示在评论列表顶部
- 显示正确的用户信息和时间
```

#### 4.2 回复评论
```
测试步骤：
1. 点击评论的"回复"按钮
2. 输入回复内容
3. 提交回复

预期结果：
- 回复成功
- 正确显示嵌套层级
- @用户名显示正确
```

#### 4.3 编辑/删除评论
```
测试步骤：
1. 找到自己的评论
2. 点击"编辑"修改内容
3. 点击"删除"删除评论

预期结果：
- 只能编辑/删除自己的评论
- 编辑后显示"已编辑"标记
- 删除后评论消失
```

### 五、用户行为追踪测试

#### 5.1 页面浏览追踪
```
测试步骤：
1. 访问多个页面
2. 在文章页面停留>30秒
3. 滚动到页面底部

预期结果：
- user_actions表记录浏览行为
- 记录阅读时长和滚动深度
- 批量发送机制正常工作
```

### 六、个人中心测试

#### 6.1 个人资料编辑
```
测试步骤：
1. 访问个人资料编辑页
2. 修改昵称、简介
3. 上传新头像
4. 保存修改

预期结果：
- 信息更新成功
- 头像上传到Supabase Storage
- 全站同步显示新信息
```

#### 6.2 评论历史
```
测试步骤：
1. 访问评论历史页面
2. 查看自己的所有评论
3. 点击评论跳转到原文

预期结果：
- 显示所有评论
- 包含回复提醒标记
- 跳转功能正常
```

## 测试工具

### 1. 手动测试
- 使用不同浏览器（Chrome、Safari、Firefox）
- 移动端测试（iOS、Android）
- 不同网络环境（3G、4G、WiFi）

### 2. 自动化测试脚本
```bash
# 运行集成测试
npm run test:integration

# 运行E2E测试
npm run test:e2e
```

### 3. 数据验证SQL
```sql
-- 验证点赞数据
SELECT COUNT(*) FROM likes WHERE content_id = 'test_article_id';

-- 验证评论数据
SELECT * FROM comments WHERE content_id = 'test_article_id' ORDER BY created_at DESC;

-- 验证用户行为
SELECT * FROM user_actions WHERE user_id = 'test_user_id' ORDER BY created_at DESC LIMIT 10;
```
